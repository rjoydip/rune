# runtime-tests/vercel

Tests `@rune/adapter-vercel` using `@vercel/fun` as a local Lambda runtime emulator.

## Approach

Mirrors how Vercel deploys Edge Functions:

1. **TypeScript source** (`src/handler.ts`) uses `@rune/adapter-vercel` + `@rune/core` + `@rune/decorators`
2. **Bundled** with Bun (ESBuild under the hood) into a single ESM file — same philosophy as Vercel's build step
3. **Lambda wrapper** (`lambda-wrapper.cjs`) adapts the bundled edge handler for `@vercel/fun`
4. **`@vercel/fun`** spawns a Node.js subprocess, loads the wrapper, and invokes it with Lambda events

## Files

| File                  | Purpose                                            |
| --------------------- | -------------------------------------------------- |
| `src/handler.ts`      | TypeScript edge handler with decorators            |
| `dist/handler.js`     | Bundled output (generated by `bun run build`)      |
| `lambda-wrapper.cjs`  | Lambda-compatible adapter for `@vercel/fun`        |
| `tests/index.test.ts` | 5 test cases                                       |
| `vercel.json`         | Vercel deployment config (edge runtime + rewrites) |

## Run

```bash
bun test
```
